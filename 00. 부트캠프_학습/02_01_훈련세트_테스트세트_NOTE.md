# 노트의 목표
**fit에 쓴 데이터로 score를 내면 100%가 나올 수 있으니 → 훈련세트/테스트세트를 분리해서 “진짜 성능”을 평가하자.**

## 데이터 준비: 길이/무게 → 2차원 입력(X), 정답(y)
```
fish_length = [...]
fish_weight = [...]

fish_data = [[l,w] for l, w in zip(fish_length, fish_weight)]   # 2차원 배열
fish_target = [1]*35 + [0]*14        # 도미=1, 빙어=0
```
**역할**
- `fish_data` : 모델이 볼 입력값 X (샘플 49개, 특성 2개: 길이/무게)
- `fish_target` : 정답 y
- 머신러닝은 “샘플(한 마리) = [특성들]” 형태로 학습함
→ 길이 리스트, 무게 리스트를 그냥 두면 모델이 바로 못 씀(특성 벡터로 묶어야 함)

---

## (실수 유도 파트) 편향된 분할을 “슬라이싱”으로 해보기
```
train_input = fish_data[:35]
train_target = fish_target[:35]
test_input = fish_data[35:]
test_target = fish_target[35:]
```
**역할**
- 앞 35개를 훈련, 뒤 14개를 테스트로 나눔
- 데이터가 앞쪽 35개 = 도미, 뒤쪽 14개 = 빙어 순서로 들어있어서
→ 훈련셋에는 도미만 몰리고, 테스트셋에는 빙어만 몰림
→ KNN이 정상적으로 학습/평가를 못 함(점수 0.0 같은 결과가 나올 수 있음)

---

## KNN 학습/평가: “편향 분할”의 결과 확인
```
from sklearn.neighbors import KNeighborsClassifier
kn = KNeighborsClassifier()

kn.fit(train_input, train_target)

```
**역할**
- `fit`: 훈련 데이터로 모델 “기준” 만들기
- `score`: 테스트 데이터로 정확도 평가
- **훈련 점수 말고 테스트 점수로 평가해야** 과적합/착시를 피할 수 있음
- 이 단계는 특히 “데이터를 잘 나누는 게 성능만큼 중요”하다는 메시지

---

## numpy로 변환 + 랜덤 셔플로 “샘플링 편향” 해결
```
import numpy as np

input_arr = np.array(fish_data)
target_arr = np.array(fish_target)

np.random.seed(42)
index = np.arange(49)
np.random.shuffle(index)
```
**역할**
- 리스트 → 넘파이 배열로 바꿔서 **배열 인덱싱** 가능하게 함
- `shuffle(index)`: 0~48 인덱스를 랜덤으로 섞어 “랜덤 분할” 기반을 만듦
- “앞은 도미, 뒤는 빙어” 같은 **정렬된 데이터**는 슬라이싱 분할에 취약함
- 랜덤 셔플은 클래스가 섞이게 만들어 편향을 줄임
- `seed(42)`는 **재현성**(실습/교재 결과를 동일하게 맞추기) 목적

---

## 배열 인덱싱으로 훈련/테스트 만들기(정답 분리 포함)
```
train_input  = input_arr[index[:35]]
train_target = target_arr[index[:35]]

test_input  = input_arr[index[35:]]
test_target = target_arr[index[35:]]
```
**역할**
- 랜덤으로 섞인 인덱스를 이용해
   - 35개는 훈련
   - 14개는 테스트
- 입력(X)과 정답(y)을 같은 인덱스 규칙으로 함께 분리
- 핵심은 “X와 y가 같이 움직여야 한다”는 것 (입력만 섞고 정답을 안 섞으면 라벨이 틀어져서 학습이 망함)

---

## 시각화로 분리 상태 확인(훈련/테스트가 섞였는지)
```
import matplotlib.pyplot as plt

plt.scatter(train_input[:,0], train_input[:,1])
plt.scatter(test_input[:,0], test_input[:,1])
plt.xlabel('length')
plt.ylabel('weight')
plt.show()
```
**역할**
- 훈련셋/테스트셋이 한쪽으로 쏠리지 않고 섞였는지 눈으로 확인
- 데이터 분할은 “숫자 출력”만으로 놓치기 쉬움
- 산점도는 편향 여부를 즉시 보여줌

---

## (정상 분할 후) 다시 학습/평가 → 점수 확인
```
kn.fit(train_input, train_target)
kn.score(test_input, test_target)
```
**역할**
- 편향이 해결된 분할로 “진짜 평가” 수행
- 같은 모델(KNN)이라도 **데이터 분할이 성능의 전제조건**이라는 걸 확인하는 단계
- “훈련=기억, 평가는 별도 데이터”라는 ML 기본기를 굳히는 파트

---

## 예측 확인: 테스트 전체/개별 샘플
```
kn.predict(test_input)
test_target

kn.predict([[25,150]])
```
**역할**
- `predict(test_input)`: 테스트셋을 통째로 예측해 실제 라벨과 비교
- `predict([[25,150]])`: 새 샘플을 넣어 분류 결과 확인(추론)
- score(정확도)는 “요약 점수”이고, predict는 “개별 결과”를 보여줌
- 특히 `[[25,150]]` 같은 케이스는 다음 챕터에서 자주 이어짐:
  - **스케일링(정규화) 필요성**
  - K값/거리 기반 모델의 민감도 등
