# 심층신경망(딥)
## 전체 코드 흐름(순서)
1. 데이터 로드
2. 정규화(스케일링)
3. 훈련/검증 분리
4. shape 확인
5. 모델 컴파일 + 학습
6. 검증 평가
7. 옵티마이저 바꿔가면서 재학습
8. 검증 성능 비교

## 코드 블록별 '역할(기능)' 
**A.데이터 로드**<br>
`(train_input, train_target), (test_input, test_target) = keras.datasets.fashion_mnist.load_data()`
- **역할** : (이미지, 라벨) 형태의 지도학습 데이터 준비
- 딥러닝 학습은 정답(`target`)으로 손실을 계산해야 해서 "X/y"가 필수

---

**B.정규화**<br>
`train_scaled = train_input / 255.0`
- **역할** : 픽셀 스케일을 줄여 학습 안정화
- 입력 스케일이 크면 경사하강이 불안정해지고 수렴이 느려질 수 있음

---

**C.훈련/검증 분리**<br>
노트에 있는 코드 
```
train_scaled, val_scaled, train_target, val_target = train_test_split(
    train_scaled, train_target, test_size=0.2, random_state=42)
```
- **역할** : 훈련용/검증용을 분리해서 **과적합 감시**
- 훈련 점수만 보면 "외운 것(과적합)"을 못 잡아서, 검증셋으로 일반화 성능을 같이 봄

---

**D.shape출력(입력형택확인)** <br>
```
print(test_input.shape, test_target.shape)   # (10000, 28, 28) (10000,)
print(train_scaled.shape, train_target.shape) # (48000, 28, 28) (48000,)
```
- **역할** : 모델에 넣기 전 입력 텐서 형태 검증
- 딥러닝은 shape가 안 맞으면 바로 에러/성능 저하. 특히 Flatten을 쓸지,reshape를 할지 결정하는 근거가 됨

---

**E.컴파일 + 학습(기본 학습 루프)** <br>
```
model.compile(loss='sparse_categorical_crossentropy', metrics=['accuracy'])
model.fit(train_scaled, train_target, epochs=5)
```
**역할**
- `compile` : "학습 규칙"을 확정(손실/지표/옵티마이저 등)
- `fit' : 실제 가중치 업데이트(학습)
- `sparse_catgorical_crossentropy': 라벨이 원-핫이 아니라 **정수 클래스(0~9)** 일 때 표준 손실

---

**F.검증평사** <br>
`model.evaluate(val_scaled, val_target)`
- **역할** : 훈련에 쓰지 않은 검증셋으로 성능 확인
- "훈련 성능 상승"이 "실전 성능 상승"인지 확인(과적합체크)

---

**G.오늘 노트의 메인: 옵티마이저 비교(하이퍼파라미터 실험)** <br>
같은 모델에서 옵티마이저만 바꿔서 학습을 반복한다.
```
model.compile(optimizer='adagrad', loss='sparse_categorical_crossentropy', metrics=['accuracy'])
model.fit(train_scaled, train_target, epochs=5)

model.compile(optimizer='rmsprop', loss='sparse_categorical_crossentropy', metrics=['accuracy'])
model.fit(train_scaled, train_target, epochs=5)

model.compile(optimizer='adam', loss='sparse_categorical_crossentropy', metrics=['accuracy'])
model.fit(train_scaled, train_target, epochs=5)
```
- **역할** : "최적화 방법(옵티마이저)"이 학습 속도/성능에 미치는 영양 비교
- 옵치마이저는 대표적인 **하이퍼파라미터** 라는 모델이 스스로 학습하지 않고 사람이 정해야 함
  
- Adagrad는 초반 성능이 상대적으로 낮게 출발하는 로그가 찍힘(정확도 ~0.5→0.8대)
- RMSprop/Adam은 더 빠르게 0.88~0.91대 학습 정확도로 올라감(로그 기준)
